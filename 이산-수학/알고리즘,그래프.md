### 알고리즘

- 문제를 해결하기 위한 절차를 기술한 것
- 누구나 정해진 절차대로 실행할 수 있어야 함
- 누구나 동일한 입력값이 주어졌을 때 이 절차대로 실행하면 동일한 출력값을 얻을 수 있어야 함

- 일정한 시간안에 출력값을 구할 수 있어야 함
<br>

#### 알고리즘의 정의

- 순서대로 정의된 절차
  - 분명한 순서가 있어야 함
  - 한 동작을 실행하면 다음에 실행할 동작이 무엇인지 분명해야 함

- 명확성
  - 모든 동작은 명확하게 정의되어야 함
  - 모든 동작은 실행 가능해야 함

- 반드시 원하는 결과가 나와야 함
- 일정한 시간 안에 실행되어야 함
<br>

#### 알고리즘 구조

1. 순차적 구조 (sequential structure)
2. 분기 구조 (branch)

3. 반복 구조 (repetition)
4. 점프 구조 (jump)
<br>

#### 알고리즘 기술 방법

- 플로우차트 (flow chart)

- 프로그램 언어의 코드
  - 특정 프로그램 언어의 문법을 알아야 함
  - 코드 수준으로 자세히 적을 필요 X
- Pseudocode (유사 코드)
  - 프로그램 언어의 코드 전 단계로 프로그램 언어들이 사용하는 구조를 그대로 사용함
<br>

#### 알고리즘 검증 (Verification)

- 수학적 귀납법
  - 몇 가지 사례(입력값)에 대하여 옳음을 보임
    - 일반적인 값
    - 특수한 입력값 (boundary condition)

### 오일러 순환과 해밀턴 순환

- 그래프
  - 정점 V(vertex)와 선 E(edge)들의 집합으로 구성

#### 오일러 정리

- 오일러 경로 (Eulerian path)
  - 그래프의 모든 연결선을 한번만 방문하는 경로
  - 2개 이상의 정점을 갖는 루프가 없는 연결 그래프에서 홀수 차수를 갖는 정점이 하나도 없거나 두 개(시작점, 끝점)만 존재해야 함

- 오일러 순환 (Eulerian cycle 혹은 Circuit)
  - 시작점과 끝점이 동일한 오일러 경로
  - 오일러 경로에서 모든 정점이 짝수 차수를 가져야 함

- 오일러 그래프
  - 오일러 순환이 존재하는 그래프

- 차수 (degree)
  - 정점 u에 접합된 연결선의 수
  - deg(u)와 같이 표기하기도 함

- 알고리즘 복잡도
  - n * 차수 ≤ n * (n-1) 
     ≤ n²
  - O(n²) 
    - 자세하게 : O(n·m) //m: 평균차수 m<n

<br>

#### 해밀톤 정리

- 해밀톤 경로 (Hamiltonian path)
  - 그래프에서 모든 정점을 정확히 한 번만 지나는 경로
- 해밀톤 순환(Hamiltonian cycle 혹은 circuit)
  - 시작점과 끝점이 같은 해밀톤 경로
- 해밀톤 순환 알고리즘
  - 해밀톤 순환을 찾는 알고리즘은 존재하지 않음 → 완전 탐색 필요
  - 탐색해야 하는 경로의 수
    - 트리의 leaf의 수
  - 트리의 높이 : n+1
  - Leaf의 수 : 2ⁿ

- 해밀톤 순환 알고리즘 복잡도
  - 알고리즘 복잡도 : O(xⁿ) (n: 정점의 수)
  - 복잡도 함수는 다항식이 아니라 지수식임
  - 유사한 복잡도 문제
    - 암호 해독
    - 바둑 (혹은, 장기, 체스 등)
    - Bin packing 등
<br>

#### TSP 알고리즘 (방문판매원 문제)

- 모든 해밀톤 순환을 찾아서 ㅎ바한 비용이 가장 작은 순환을 선택
- 단순 TSP 알고리즘 (최적해 보장 X)
  1. 하나의 정점을 선택하여 출발점으로 한다.
  2. 이 정점에 연결된 연결선의 비용이 가장 작은 정점을 선택한다.
  3. 이 정점에서부터 아직 선택되지 않은 정점들 중에서 연결선의 비용이 가장 작은 정점을 선택한다.
  4. 모든 정점을 선택할 때까지 2와 3의 절차를 반복한다.

### 그래프 기본 용어

- 정점 V(vertex)와 선 E(edge)들의 집합으로 구성

- 임의의 연결선 e =(u, v)에 대해서 정점 u와 정점 v는 서로 인접(adjacent)했다고 하며,
  e는 정점 u와 정점 v에 접합(incident)한다고 한다.

- 루프(loop) : 연결선의 두 끝점이 같은 정점인 연결선
- 다중 연결선 : 정점의 연결선이 두 개 이상일 때
- 단순 그래프 (simple graph) : 루프나 다중 연결선이 없는 그래프

- 차수(degree) : 정점에 접합된 연결선의 수 (정점 u의 차수)
  - deg(u)와 같이 표시
  - 그래프에서 모든 정점의 차수의 합은 모든 연결선 수의 2배

- 연결(connected) : 두 정점 사이에 연결선이 존재할 경우



- 길이(length) : 두 정점의 경로를 구성하는 간선의 수
- 거리(distance) : 두 정점 간의 최단 경로의 길이
- 닫힌 경로(closed path) : 시작점과 끝 점이 같은 경로
- 순환(cycle 혹은 circuit) : 3개 이상의 간선을 갖는 경로에서 어떤간선도 중복되지 않는 닫힌 경로

- 면(face) : 연결선에 따라 구분된 영역

<br>

#### 그래프의 종류

- 부분 그래프 (subgraph) 
  - 어떤 그래프가 있을 때 그래프에 포함되는 일부 정점과 간선으로만 그린 그래프

- 동형 그래프 (isomorphic graph)
  - 두 그래프가 똑같은 정점과 간선으로 구성되어 있을 경우
    - 정점과 간선이 같다면 다양한 형태의 동형 그래프 존재할 수 있음

- 완전 그래프 (complete graph)
  - 모든 정점 사이에 간선이 존재하는 그래프
  - K_m으로 표시 (m은 정점의 총 수)

- 이분 그래프 (bipartite graph)
  - 정점들이 두 집합으로 나눠진 그래프
  - 연결선이 다른 집합의 정점들과만 연결된 그래프
  - 같은 집합인 정점들끼리는 연결선 존재 X
- 완전 이분 그래프
  - 이분 그래프에서 두 집합 사이의 모든 정점들이 간선을 가지는 그래프

- 평면 그래프 (planar graph)
  - 간선들이 서로 교차하지 않고 평면상으로 그릴 수 있는 그래프
  - PCB에서 사용
- 방향 그래프 (directed graph, digraph)
  - 그래프에서 간선의 두 정점이 순서쌍일 때
